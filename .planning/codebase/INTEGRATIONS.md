# External Integrations

**Analysis Date:** 2026-03-06

## APIs & External Services

**MyJDownloader Web API:**
- API Endpoint: `https://api.jdownloader.org`
- Purpose: Authentication, device discovery, link submission
- SDK/Client: Custom implementation in `vendor/js/jdapi.js` (RequireJS module)
- Auth: OAuth2-based via `myjdClientFactory`
- Usage: Offscreen document loads API via RequireJS in `offscreen.js`

**JDownloader Local Instance:**
- Endpoint: `http://127.0.0.1:9666/` or `http://localhost:9666/`
- Purpose: Click'N'Load (CNL) interception, CAPTCHA submission, link addition
- Protocol: HTTP GET/POST with URL parameters
- Key paths:
  - `/crossdomain.xml` - CORS check
  - `/flash/addcrypted2` - CNL crypto payload submission
  - `/flash/add` - Plain link submission
  - `/captcha/...` - CAPTCHA job endpoint (intercepted by extension)
- Custom header: `X-Myjd-Appkey: myjd-captcha-helper-{version}`

**CAPTCHA Services (via native helper WebView2):**
- reCAPTCHA v2 (Checkbox) - `siteKey`, user interaction
- reCAPTCHA v3 (Invisible) - `siteKey`, score-based
- reCAPTCHA Enterprise - Enhanced protection variant
- hCaptcha - Open-source alternative
- Implementation: `captcha-helper/src/html.rs` generates HTML with iframe injection
- Token submission: HTTP POST back to JDownloader callback URL

**My.JDownloader Web Interface:**
- Endpoint: `https://my.jdownloader.org/`
- Purpose: Web-based account management, device control
- Usage: CAPTCHA token submission via postMessage for web interface mode
- Content script: `webinterfaceEnhancer.js` runs on `https://my.jdownloader.org/*`

## Webhooks & Callbacks

**Incoming:**
- Content script message handlers: Tab-to-background communication
- `chrome.runtime.onMessage` listeners for:
  - Toolbar link updates
  - CNL capture from content scripts
  - Offscreen document responses
  - Connection state changes

**Outgoing:**
- HTTP GET to JDownloader callback URLs:
  - Format: `http://127.0.0.1:9666/captcha/{captchaType}/{hoster}?id={id}&do=solve&response={token}`
  - Used by: `scripts/services/CaptchaNativeService.js` (line 60)
  - Used by: `scripts/services/Rc2Service.js` (line 86)

**Native Messaging Callbacks:**
- Extension → Native Helper: `chrome.runtime.sendNativeMessage()`
  - Host: `org.jdownloader.captcha_helper`
  - Payload: JSON with action, siteKey, callbackUrl, captchaUrl, etc.

- Native Helper → Extension: Stdout with message envelope
  - Statuses: `opened`, `solved`, `skipped`, `timeout`, `cancelled`, `error`
  - Response format: 4-byte length + JSON payload

**postMessage to Tabs:**
- Extension → my.jdownloader.org tab: Deliver CAPTCHA token
- Message format: `{ name: 'response', type: 'myjdrc2', data: { captchaId, token } }`
