<bugs_found>

### BUG 1 (CRITICAL): Extension cannot detect CAPTCHAs without my.jdownloader.org open

**Problem:** The extension has ZERO independent CAPTCHA detection. The ONLY trigger path requires my.jdownloader.org to be open in a browser tab:

1. `webinterfaceEnhancer.js` (content script on my.jdownloader.org) detects a CAPTCHA signal from the web interface
2. The my.jdownloader.org page navigates a tab with `#rc2jdt&c=CAPTCHAID` in the URL hash
3. `Rc2Service.js` `chrome.tabs.onUpdated` listener (line 290-324) detects `#rc2jdt` hash
4. Rc2Service queries MyJD API for job details, calls `onWebInterfaceCaptchaJobFound()`
5. Service worker prepares CAPTCHA tab

**Without my.jdownloader.org open, step 1-2 never happens.** CAPTCHAs silently get ignored. The user explicitly confirmed: "That is not how it used to work; that is not how this is supposed to work." The old MV2 extension detected CAPTCHAs independently.

**Fix needed:** Add CAPTCHA job polling. When the extension is logged into MyJD, periodically call the MyJD API to check for pending CAPTCHA jobs. When found, trigger `onWebInterfaceCaptchaJobFound()` directly (skipping steps 1-2). The API endpoints already exist in Rc2Service: `/captcha/getCaptchaJob` and `/captcha/get` (lines 300-311). Use `myjdClientFactory` and `myjdDeviceClientFactory` (already injected into Rc2Service).

**Key files:**
- `scripts/services/Rc2Service.js` lines 290-324 — `chrome.tabs.onUpdated` listener (the ONLY current trigger)
- `scripts/services/Rc2Service.js` lines 54-72 — `onWebInterfaceCaptchaJobFound()` (what polling should call)
- `scripts/services/Rc2Service.js` lines 300-311 — MyJD API calls for CAPTCHA job
- `contentscripts/webinterfaceEnhancer.js` — current trigger source (will become optional after fix)
- `background.js` line 615+ — `myjd-prepare-captcha-tab` handler

### BUG 2: `captchaSolverContentscript.js` injecting skip buttons into random websites

**Problem:** On NitroFlare's free download page (URL: `https://nitroflare.com/view/.../free`, NO `#rc2jdt` hash), the extension's skip buttons ("Skip nitroflare.com CAPTCHAs", "Skip Package", "Skip All", "Skip This") and countdown timer appeared at the bottom of the page. Verified via eval: `body.id` was empty (not `myjd-captcha-body`), `captchaContainer` not found — confirming `myjdCaptchaSolver.js` did NOT activate. The buttons come from `captchaSolverContentscript.js`.

**The script** is registered in `manifest.json` lines 52-57 to run on `*://*/*` at `document_end`. It's supposed to only enhance JD localhost CAPTCHA pages (`http://127.0.0.1:\d+/captcha/...`), but it's injecting UI into ANY page that has a CAPTCHA widget.

**Fix needed:** Add URL gate to `captchaSolverContentscript.js` — check that `location.href` matches `http://127.0.0.1:\d+/captcha/(recaptchav(2|3)|hcaptcha)/` before doing anything. Similar to how `myjdCaptchaSolver.js` has its `#rc2jdt` hash gate at line 5.

**Key files:**
- `contentscripts/captchaSolverContentscript.js` — the leaking script (needs URL gate)
- `manifest.json` lines 52-57 — its registration
- `contentscripts/myjdCaptchaSolver.js` line 5 — example of proper gate pattern
</bugs_found>

<chrome_cdp_setup>
Chrome must be launched with specific flags for browser automation. Here's the exact procedure:

1. Kill ALL Chrome processes first (required — old processes block debug port silently):
   `taskkill //F //IM chrome.exe`

2. Wait 3 seconds, then launch via PowerShell:
   ```powershell
   Start-Process 'C:\Program Files\Google\Chrome\Application\chrome.exe' -ArgumentList '--remote-debugging-port=9222','--remote-allow-origins=*','--user-data-dir=C:\Users\anthony\AppData\Local\Google\Chrome\User Data','--profile-directory=Default','--load-extension=C:\Users\anthony\jdownloader-extension-manifestv3'
   ```

3. Verify port is open: `netstat -ano | findstr "9222"` should show LISTENING

4. User must manually load extension and sign into MyJD through extension popup

5. Chrome MCP tool (`mcp__plugin_superpowers-chrome_chrome__use_browser`) connects via CDP on port 9222
   - Available actions: navigate, click, type, eval, screenshot, list_tabs, new_tab, close_tab, keyboard_press, await_element, await_text
   - CANNOT right-click or trigger Chrome context menus — user must do right-clicks manually
   - Screenshots auto-save to `C:\Users\anthony\AppData\Local\superpowers\browser\YYYY-MM-DD\session-*\`
   - Use `Read` tool on `.png` files to view screenshots

**Important notes:**
- User's Chrome profile is "Default" (NOT "Profile 3")
- `--load-extension` flag loads the extension from the repo directory
- `chrome://extensions/` navigation kills the CDP connection — do NOT navigate there
- The first `list_tabs` call after launch may fail; retry after a few seconds
