**2. Add three new message handlers inside the existing `chrome.runtime.onMessage.addListener` callback, BEFORE the "Default: acknowledge unknown actions" block (around line 571):**

Handler for `captcha-tab-detected`:
- Extract data from `request.data` (callbackUrl, captchaType, hoster, captchaId)
- Store in `activeCaptchaTabs[sender.tab.id]` with those fields plus `detectedAt: Date.now()`
- `sendResponse({ status: 'ok' })`
- `return true`

Handler for `captcha-solved` (CAP-03, CAP-04):
- Remove tab from `activeCaptchaTabs` if `sender.tab` exists
- Create `XMLHttpRequest`, open GET to `request.data.callbackUrl + '&do=solve&response=' + encodeURIComponent(request.data.token)`
- Set header `X-Myjd-Appkey` to `'webextension-' + chrome.runtime.getManifest().version`
- Set timeout to 10000ms (10 seconds)
- On `onload`: close the CAPTCHA tab after 2-second delay per user decision (auto-close, no success message): `setTimeout(function() { chrome.tabs.remove(sender.tab.id, function() { if (chrome.runtime.lastError) { /* ignore */ } }); }, 2000)`
- On `ontimeout`: log error but don't retry
- `sendResponse({ status: 'ok' })`
- `return true`
- NOTE: Use `encodeURIComponent` on the token to prevent URL injection via special characters in the token string.

Handler for `captcha-skip`:
- Remove tab from `activeCaptchaTabs` if `sender.tab` exists
- Create `XMLHttpRequest`, open GET to `request.data.callbackUrl + '&do=skip&skiptype=' + request.data.skipType`
- Set header `X-Myjd-Appkey` to `'webextension-' + chrome.runtime.getManifest().version`
- Set timeout to 10000ms
- On `onload`: close CAPTCHA tab after 2-second delay (same as solve)
- `sendResponse({ status: 'ok' })`
- `return true`

**3. Modify the existing `chrome.tabs.onRemoved` listener (line 621-624) to also handle CAPTCHA tab close (CAP-07):**

The current listener is:
```javascript
chrome.tabs.onRemoved.addListener((tabId) => {
  delete requestQueue[String(tabId)];
  persistQueue();
});
```

Add CAPTCHA tab handling to this same listener:
```javascript
chrome.tabs.onRemoved.addListener((tabId) => {
  // Existing: clean up request queue
  delete requestQueue[String(tabId)];
  persistQueue();

  // CAPTCHA: send skip on tab close (CAP-07)
  if (activeCaptchaTabs[tabId]) {
    var info = activeCaptchaTabs[tabId];
    delete activeCaptchaTabs[tabId];
    var httpRequest = new XMLHttpRequest();
    httpRequest.open('GET', info.callbackUrl + '&do=skip&skiptype=single', true);
    httpRequest.setRequestHeader('X-Myjd-Appkey', 'webextension-' + chrome.runtime.getManifest().version);
    httpRequest.timeout = 10000;
    httpRequest.send();
    console.log('Background: CAPTCHA tab closed, sent skip(single) for', info.hoster);
  }
});
```

Note: Use `skiptype=single` for tab close (Claude's discretion per CONTEXT.md -- single is less aggressive, consistent with timeout behavior). Delete the entry from activeCaptchaTabs BEFORE sending the HTTP request to prevent the captcha-skip handler from also trying to skip if a message arrives simultaneously.
