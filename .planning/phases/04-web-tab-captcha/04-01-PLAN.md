  <name>Task 1: Add JD protocol callbacks to captchaSolverContentscript.js</name>

  <action>
Add three new functions to the existing CAPTCHA content script, called ONLY when `isJdLocalhost` is true (protocol callbacks are meaningless on external sites). Add them after the existing `startCountdown` function.

1. **canClose polling** (`startCanClosePolling(callbackUrl)`):
   - Create a new `var canCloseHandle` at the top alongside `pollingHandle` and `countdownHandle`
   - Use `setInterval` at 1000ms (1 second)
   - Each tick: create `XMLHttpRequest`, GET `callbackUrl + '&do=canClose'`
   - Set header `X-Myjd-Appkey` to `'webextension-' + chrome.runtime.getManifest().version`
   - Set `xhr.timeout = 5000`
   - On `readyState === 4 && status === 200`: if `xhr.response === 'true'`, clear ALL intervals (canCloseHandle, pollingHandle, countdownHandle), then close tab via `window.close()` -- no delay, no message (per CONTEXT.md)
   - If `window.close()` does not work (content scripts cannot always close tabs), send a message to the service worker: `chrome.runtime.sendMessage({ action: 'captcha-can-close' })` as a fallback, and have the service worker close the tab
   - Call this function after `startCountdown()` in the main flow, only when `isJdLocalhost`

2. **loaded event** (`sendLoadedEvent(callbackUrl)`):
   - Find the CAPTCHA widget element using: `document.querySelector('iframe[src*="recaptcha"], iframe[src*="hcaptcha"], .g-recaptcha, .h-captcha')`
   - If not found immediately, retry with a 500ms `setTimeout` up to 10 times (the CAPTCHA iframe may take time to render)
   - Once found, get `getBoundingClientRect()` for the element
   - Build params string: `x`, `y` (window.screenX/screenLeft, window.screenY/screenTop), `w`, `h` (window.outerWidth/outerHeight), `vw`, `vh` (window.innerWidth/innerHeight), `eleft`, `etop`, `ew`, `eh` (from bounding rect), `dpi` (window.devicePixelRatio)
   - XMLHttpRequest GET to `callbackUrl + '&do=loaded&' + params`
   - Set `X-Myjd-Appkey` header, timeout 5000
   - Call this function after `startCountdown()` in the main flow, only when `isJdLocalhost`

3. **mouse-move reporting** (`startMouseMoveReporting(callbackUrl)`):
   - Add `mousemove` event listener on `document`
   - Throttle to 3 seconds (track `var lastMouseMoveTime = 0` at top with other vars)
   - On throttled mouse-move: XMLHttpRequest GET to `callbackUrl + '&do=canClose&useractive=true&ts=' + Date.now()`
   - Set `X-Myjd-Appkey` header, timeout 5000
   - Call this function after `sendLoadedEvent()` in the main flow, only when `isJdLocalhost`

4. **Update beforeunload cleanup** to also clear `canCloseHandle`

5. **Add captcha-can-close handler in background.js**: Add a message handler for `action: 'captcha-can-close'` in `background.js` that closes `sender.tab.id` and cleans up `activeCaptchaTabs`. This is a fallback for when `window.close()` fails in the content script.

All code must use `var` declarations (not let/const) and function statements, matching the existing IIFE style. Use `XMLHttpRequest` (not fetch) to match the existing codebase pattern.
