  <action>
Create `contentscripts/captchaSolverContentscript.js` as an IIFE following the pattern in `webinterfaceEnhancer.js`.

**URL detection (CAP-01):**
- First line after IIFE open: check `window.location.pathname` against `/captcha/(recaptchav2|recaptchav3|hcaptcha)/` regex. Return immediately if no match (prevents activation on CNL and other localhost pages per Pitfall 5).
- Extract from URL: `captchaType` (path segment 2), `hoster` (path segment 3, URI-decoded), `captchaId` (from `?id=` query param).
- Use `window.location.href` as the `callbackUrl` (the URL IS the callback -- Pitfall 4).
- Do NOT hardcode port 9666; use whatever port is in the URL.

**Registration message:**
- Immediately send `captcha-tab-detected` message to service worker via `chrome.runtime.sendMessage()` with `{action: 'captcha-tab-detected', data: {callbackUrl, captchaType, hoster, captchaId}}`.

**Token polling (CAP-02, CAP-10):**
- Create `startTokenPolling(callbackUrl)` function.
- Use `setInterval` at 500ms.
- Poll both reCAPTCHA and hCaptcha textareas:
  - `document.querySelectorAll('textarea[id^="g-recaptcha-response"]')` for reCAPTCHA v2/v3 (handles multiple instances per Pitfall 2).
  - `document.querySelectorAll('textarea[name="h-captcha-response"]')` for hCaptcha.
- Token is valid when `.value` exists and `.value.length > 30`.
- On token found: `clearInterval`, send `{action: 'captcha-solved', data: {token, callbackUrl}}` via `chrome.runtime.sendMessage()`.
- Store interval handle so countdown can clear it on timeout.

**Skip buttons (CAP-05):**
- Create `injectSkipButtons(callbackUrl, hoster)` function.
- Create a container `div#myjd-captcha-controls` with flexbox layout (centered, wrap, 16px top margin).
- Four buttons with extension-themed styling (blue/white: `#2196F3` border+text on white background, 4px border-radius, 8px 16px padding, 13px font-size, pointer cursor).
- Button labels: "Skip {hoster} CAPTCHAs" (hoster type), "Skip Package", "Skip All", "Skip This" (single type).
- Use event delegation: single `click` listener on container, read `e.target.dataset.skipType`.
- On click: send `{action: 'captcha-skip', data: {callbackUrl, skipType}}` via `chrome.runtime.sendMessage()`.
- Add hover effect via mouseenter/mouseleave listeners (background: #2196F3, color: #fff on hover).
- Place buttons by appending container to `document.body`. The CAPTCHA widget renders in the page's existing layout; buttons go below it.

**Countdown timer (CAP-06):**
- Create `startCountdown(callbackUrl)` function.
- 5-minute timeout (300000ms), hardcoded per user decision.
- Create `div#myjd-countdown` element, centered, 14px font, `#666` color, 12px top margin.
- `setInterval` at 1000ms updating display as `Time remaining: M:SS`.
- Visual urgency: when remaining < 60 seconds, change color to `#f44336` (red) and add `fontWeight: 'bold'`.
- On expiry: clear both countdown interval AND token polling interval, set text to "Timed out - skipping...", send `{action: 'captcha-skip', data: {callbackUrl, skipType: 'single'}}` (skip single per user decision).

**Cleanup on unload:**
- Add `window.addEventListener('beforeunload', ...)` to clear both intervals (token polling and countdown). This prevents memory leaks.

**Execution order in IIFE:**
1. URL pattern check (exit early if no match)
2. Extract metadata (captchaType, hoster, captchaId, callbackUrl)
3. Send captcha-tab-detected message
4. Call injectSkipButtons(callbackUrl, hoster)
5. Call startTokenPolling(callbackUrl) -- save handle
6. Call startCountdown(callbackUrl) -- pass polling handle for cleanup

Use `var` (not `let`/`const`) for consistency with existing content scripts in the project. Use `function` declarations (not arrow functions) for the same reason.

```json

  "all_frames": false,
  "js": ["contentscripts/captchaSolverContentscript.js"],
```
