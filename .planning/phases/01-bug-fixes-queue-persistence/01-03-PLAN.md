**Fix 1 — notifyContentScript() message routing (lines 84-107):**

Replace the entire `notifyContentScript` function with this logic:
- Send `open-in-page-toolbar` via `chrome.tabs.sendMessage(tabId, ...)` — this correctly reaches the content script which shows/creates the iframe.
- Send `link-info-update` via `chrome.runtime.sendMessage(...)` — this reaches ALL extension contexts including the toolbar iframe where ToolbarController listens via `chrome.runtime.onMessage`. Include `tabId` in the message so ToolbarController's filter (`msg.tabId == instanceId`) works.
- Fire BOTH messages simultaneously (no `.then()` chaining). The open-in-page-toolbar going to the content script and link-info-update going to the toolbar iframe are independent — they do not need to wait for each other.
- On the content script injection path (`.catch()` when content script not yet loaded):
  1. Inject via `chrome.scripting.executeScript`
  2. After injection, send `open-in-page-toolbar` via `chrome.tabs.sendMessage`
  3. Send `link-info-update` via `chrome.runtime.sendMessage` — but use a 500ms delay for this path because the iframe needs time to load and Angular needs to bootstrap before the ToolbarController's `chrome.runtime.onMessage` listener is registered. Note: the toolbar's `init()` -> `updateLinks()` will also fetch the queue on load, so the delayed `link-info-update` is a safety net for race conditions, not the primary data fetch.

The corrected function:
```javascript
function notifyContentScript(tabId) {
  // open-in-page-toolbar -> content script (shows/creates iframe)
  chrome.tabs.sendMessage(tabId, { action: "open-in-page-toolbar", tabId: tabId })
    .catch(() => {
      // Content script not loaded — inject it, then retry
      console.log("Background: Injecting toolbar content script into tab", tabId);
      chrome.scripting.executeScript({
        target: { tabId: tabId },
        files: ['contentscripts/toolbarContentscript.js']
      }).then(() => {
        setTimeout(() => {
          chrome.tabs.sendMessage(tabId, { action: "open-in-page-toolbar", tabId: tabId }).catch(e => {
            console.error("Background: Failed to notify content script after injection:", e);
          });
        }, 100);
        // Delayed link-info-update for freshly created iframe (Angular bootstrap time)
        setTimeout(() => {
          chrome.runtime.sendMessage({ action: "link-info-update", tabId: tabId }).catch(() => {});
        }, 500);
      }).catch(e => {
        console.error("Background: Failed to inject toolbar content script:", e);
      });
    });

  // link-info-update -> all extension contexts (toolbar iframe's ToolbarController listens via chrome.runtime.onMessage)
  chrome.runtime.sendMessage({ action: "link-info-update", tabId: tabId }).catch(() => {});
}
```

Change the function signature to `async function addLinkToRequestQueue(link, tab)` and add `await queueReady;` as the FIRST line before any access to `requestQueue`. This ensures the in-memory queue is restored from session storage before the duplicate check runs.
