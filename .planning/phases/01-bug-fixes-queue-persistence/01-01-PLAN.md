<objective>
Migrate the in-memory `requestQueue` in background.js to a write-through cache backed by `chrome.storage.session`, so link state survives service worker termination/restart cycles.

Purpose: Service workers terminate after 30 seconds of inactivity. Without persistence, right-clicked links are lost between termination and the next user action. This is the foundation for reliable link handling in all subsequent phases.
Output: Updated background.js with write-through persistence, Jest test file, and Chrome API mocks in jest.setup.js.
</objective>

1. **Add constants and restore function** (after line 28, after `let requestIDCounter = 0`):
   ```javascript
   const QUEUE_STORAGE_KEY = 'myjd_request_queue';

   async function restoreRequestQueue() {
       try {
           const result = await chrome.storage.session.get(QUEUE_STORAGE_KEY);
           if (result[QUEUE_STORAGE_KEY]) {
               requestQueue = result[QUEUE_STORAGE_KEY];
               console.log('Background: Restored request queue from session storage');
           }
       } catch (e) {
           console.error('Background: Failed to restore queue:', e);
       }
   }

   function persistQueue() {
       chrome.storage.session.set({ [QUEUE_STORAGE_KEY]: requestQueue }).catch(err => {
           console.error('Background: Failed to persist queue:', err);
       });
   }

   let queueReady = restoreRequestQueue();
   ```
