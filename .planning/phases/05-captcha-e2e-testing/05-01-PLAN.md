  <action>
Create a comprehensive E2E test script at `.planning/phases/05-captcha-e2e-testing/05-E2E-TEST-SCRIPT.md` that Plan 02 will execute during live testing. The script must cover the full MYJD remote CAPTCHA flow.

**Structure the test script with these sections:**

### Prerequisites Checklist
- Extension loaded at chrome://extensions and error-free
- Extension logged in to MyJD (verify via popup — connected device shown)
- my.jdownloader.org open in a browser tab, logged in
- JDownloader running on NAS, connected to MyJD account
- JDownloader Settings > Captchas > Browser Solver enabled
- No downloads currently running in JD

### Test 1: MYJD Remote CAPTCHA Flow — Full E2E
Steps:
1. Research and find a CAPTCHA-gated download link (try Rapidgator, Nitroflare, Turbobit — user is premium on pixeldrain, skip it)
2. Add the download link to JDownloader (via extension right-click or JD directly)
3. Wait for JDownloader to encounter the CAPTCHA
4. Verify: Extension detects CAPTCHA — new browser tab opens automatically
5. Verify: Tab navigates to target domain with `#rc2jdt` hash
6. Verify: CAPTCHA widget renders (reCAPTCHA or hCaptcha depending on hoster)
7. **User action**: Solve the CAPTCHA
8. Verify: Tab auto-closes after ~2 seconds
9. Verify: JDownloader shows download progressing (user confirms)

Expected service worker console log sequence:
```
"Background message: myjd-prepare-captcha-tab from: extension"
"Background: MYJD CAPTCHA tab prepared: <tabId> <hoster>"
"Background message: captcha-solved from: tab:<tabId>"
```

### Test 2: CAPTCHA Tab Close = Skip
Steps:
1. Trigger another CAPTCHA (add another download link or wait for next one)
2. Wait for CAPTCHA tab to open
3. Close the CAPTCHA tab manually (do NOT solve)
4. Verify: JDownloader shows CAPTCHA as skipped
5. Verify: Service worker console shows `"Background: CAPTCHA tab closed, sent tab-closed for <hoster>"`

### Test 3: Extension State Verification
Between tests, verify extension internal state via Chrome DevTools:
- `chrome.storage.session.get('myjd_captcha_job', console.log)` — should be empty after tab closes
- `chrome.declarativeNetRequest.getSessionRules(rules => console.log(rules))` — CSP rules should be cleaned up
- Service worker console should show no errors

### Pass/Fail Criteria
- **Pass**: CAPTCHA tab appears, user solves, token submits, JDownloader starts download
- **Fail**: Any break in the chain (tab doesn't open, widget doesn't render, token doesn't submit, JD doesn't receive token)

### On Failure Protocol
- Capture service worker console output
- Check chrome.storage.session state
- Check CSP rules
- Note the failure point in the chain
- Debug and fix inline — the goal is a working flow, not just a test report

### File Hosters to Try (in priority order)
1. Rapidgator — most likely to trigger reCAPTCHA v2 for free users
2. Nitroflare — reCAPTCHA v2 or hCaptcha
3. Turbobit — reCAPTCHA
4. DDownload — reCAPTCHA/hCaptcha
5. Uploaded.net — reCAPTCHA v2
(Skip pixeldrain — user is premium)

### CAPTCHA Type Coverage (TEST-03)
- Document which CAPTCHA types were encountered during testing
- Goal: at least reCAPTCHA v2 confirmed; hCaptcha if naturally encountered
- The code supports reCAPTCHA v2, v3, hCaptcha, and reCAPTCHA Enterprise — test confirms whichever JD encounters
